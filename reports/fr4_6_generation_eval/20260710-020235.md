K8s HPA 基于自定义指标扩缩容的核心流程是：先部署 Prometheus 或其他 Metrics Server 收集业务指标，再通过 KEDA 或原生 Custom Metrics API 将指标暴露给 HPA。标准方式是安装 prometheus-adapter，在 adapter 的配置中定义 custom.metrics.k8s.io API，让 HPA 能读取自定义指标。创建 HPA 时指定 type: Custom，metrics 字段引用自定义指标名称和目标值即可。KEDA 作为更现代的方案，通过 ScaledObject 绑定 Scaler（如 Kafka consumer lag、Redis 队列长度等），内部自动与 HPA 协作完成扩缩容。关键点在于指标必须在 custom.metrics.k8s.io API 中可查询，且 Deployment 的 selector 要与 HPA 正确匹配。
