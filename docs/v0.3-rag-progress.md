### 实现细节

- 分块器是纯函数，不读写文件，便于单测和后续索引脚本复用。
- 标题只识别 `#` 到 `###`，并维护标题栈生成 `title_path`，例如 `Agent 架构设计 > 非目标 > 测试策略`。
- 空章节不会进入索引，避免只有标题、没有正文的内容污染检索结果。
- 超长内容使用固定滑窗切分，默认 `CHUNK_SIZE=512`、`CHUNK_OVERLAP=50`。
- 如果超长内容属于某个标题章节，每个切出的块都会重复带上该标题行，保证 embedding 时保留章节语义。
- `KnowledgeChunk.chunk_id` 使用 `{source}#{chunk_index}`，保证同一输入重复分块时 ID 确定。
- `EmbeddingConfig` 复用 `OPENAI_API_KEY` 和 `OPENAI_BASE_URL`，新增 `EMBEDDING_MODEL`。
- `EmbeddingClient.embed_texts()` 支持批量文本；空列表直接返回空列表，不触发 API 调用。
- OpenAI SDK 抛出的异常统一包装成 `EmbeddingError`，后续工具层可以转换为 `embedding_failed`。

### 实现细节

- Chroma 集合名固定为 `learning_notes`，创建时配置 `metadata={"hnsw:space": "cosine"}`。
- `KnowledgeRepository.rebuild()` 会先删除旧集合再创建新集合，保证全量重建幂等。
- `KnowledgeRepository.search()` 把 Chroma 的 cosine distance 统一转换为 `similarity = 1 - distance`，工具层不接触 Chroma 原始距离。
- `search_learning_notes()` 先检查 query，再检查索引是否存在，再调用 embedding，最后按 `SIMILARITY_THRESHOLD=0.35` 过滤结果。
- 工具成功返回同时包含 `results` 和 `items`：`results` 表达 RAG 语义，`items` 兼容现有 ReAct trace 预览，避免修改循环。
- 工具结果里的每条 item 带 `title`、`content`、`source`、`title_path`、`similarity`，并额外提供 `match_score` 和 `raw_text_excerpt` 给调试面板。
- prompt 新增 `KNOWLEDGE_TOOL_PROMPT`，所有 persona 共享；引用笔记时要求使用 `（来源：文件名）` 格式。
- 建索引脚本按 `docs/**/*.md` 排序扫描，逐文件分块，按 `EMBEDDING_BATCH_SIZE=32` 批量生成 embedding 后全量重建 Chroma。
