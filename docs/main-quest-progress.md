# Tutor Agent API 主线任务进度表

这份文档是你的从 0 到 1 后端学习任务日志。它不是拿来一次性读完的，而是拿来边做边打勾的。

项目主线：做出一个 **AI 学习辅导 Agent 后端 API**。

学习主线：通过这个项目学会后端开发的基础流程，并逐步理解 AI 辅助全栈开发。

使用方式：

- 每完成一个任务，就把 `[ ]` 改成 `[x]`。
- 每个阶段都要先跑起来，再解释清楚，再进入下一阶段。
- 遇到报错时，不要急着跳过；把报错复制给 AI，让 AI 带你定位。
- 每个阶段结束后，至少写 3 句话复盘。
- 如果你无法解释自己刚写的代码，就不算真正完成。

通关原则：

- 能运行
- 能测试
- 能解释
- 能复盘

## AI 辅导提示词模板

当你准备开始一个阶段时，可以把下面这段发给 AI：

```text
我要开始 Tutor Agent API 的【阶段 X：阶段名称】。

请你作为我的后端开发导师，用新手能理解的方式辅导我。

要求：
1. 先解释这个阶段的目标。
2. 再告诉我今天只需要完成哪 1-3 个小任务。
3. 每一步都先解释为什么做，再给代码或命令。
4. 我执行后会把结果发给你，你再判断下一步。
5. 不要一次性给我太多内容。

这是这个阶段的任务清单：
【把阶段任务复制到这里】
```

当你遇到报错时，可以把下面这段发给 AI：

```text
我在 Tutor Agent API 项目里遇到了报错。

当前阶段：
我刚才想做的事情：
我执行的命令：
完整报错：

请你不要直接猜答案。
请先帮我判断这是环境问题、依赖问题、配置问题、代码问题，还是模型 API 问题。
然后一步一步带我定位。
```

当你完成一个阶段时，可以把下面这段发给 AI：

```text
我完成了 Tutor Agent API 的【阶段 X：阶段名称】。

请你像导师一样检查我是否真的理解了。
请问我 5 个问题：
1. 2 个概念理解问题
2. 2 个代码流程问题
3. 1 个扩展思考问题

如果我的回答不清楚，请纠正我。
如果我通过了，请帮我进入下一阶段。
```
