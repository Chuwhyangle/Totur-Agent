# AI 协作指南

## 1. 协作定位

在 Tutor Agent API 项目中，AI 不是单纯的代码生成器，而是：

- 后端开发导师
- 结对编程伙伴
- 代码审查员
- 调试教练
- 复盘提问者

学习者不是旁观者，而是项目负责人。AI 可以带路，但学习者要负责理解、执行、提问、复盘和决定下一步。

## 2. 协作原则

每个阶段都按小任务推进，不要求 AI 一次性生成完整项目。

推荐循环：

```text
理解目标
-> 做一个小任务
-> 运行验证
-> 解释结果
-> 打勾记录
-> 进入下一步
```

每次 AI 给出代码后，学习者都要做三件事：

```text
1. 运行它
2. 看结果
3. 用自己的话解释它
```

如果解释不出来，就不算真正完成。

## 3. AI 的四种工作模式

### 3.1 学习模式

用于不懂概念时。

提示词模板：

```text
请用新手能理解的方式解释【概念】。

要求：
1. 先用生活类比解释。
2. 再用后端术语解释。
3. 最后结合 Tutor Agent API 项目说明它会用在哪里。
4. 最后问我 1 个小问题，检查我是否理解。
```

示例：

```text
请用新手能理解的方式解释什么是 FastAPI 路由。

要求：
1. 先用生活类比解释。
2. 再用后端术语解释。
3. 最后结合 Tutor Agent API 项目说明它会用在哪里。
4. 最后问我 1 个小问题，检查我是否理解。
```

### 3.2 实现模式

用于准备写代码时。

提示词模板：

```text
请带我实现【具体任务】。

要求：
1. 先说明这个任务的作用。
2. 再告诉我应该修改哪些文件。
3. 每个文件为什么要改，都要解释。
4. 给出代码后，逐行解释关键部分。
5. 最后告诉我如何运行验证。
6. 不要一次性扩展无关功能。
```

示例：

```text
请带我实现 GET /health 接口。

要求：
1. 先说明这个接口的作用。
2. 再告诉我应该修改哪个文件。
3. 给出代码后，逐行解释关键部分。
4. 最后告诉我如何运行验证。
```

### 3.3 调试模式

用于遇到报错时。

提示词模板：

```text
我在 Tutor Agent API 项目里遇到了报错。

当前阶段：
我刚才想做的事情：
我执行的命令：
完整报错：

请你不要直接猜答案。
请先帮我判断这是环境问题、依赖问题、配置问题、代码问题，还是模型 API 问题。
然后一步一步带我定位。
每一步只让我做一个检查。
```

### 3.4 复盘模式

用于阶段完成后检查自己是否真的理解。

提示词模板：

```text
我完成了 Tutor Agent API 的【阶段 X：阶段名称】。

请你像导师一样检查我是否真的理解了。
请问我 5 个问题：
1. 2 个概念理解问题
2. 2 个代码流程问题
3. 1 个扩展思考问题

如果我的回答不清楚，请纠正我。
如果我通过了，请帮我进入下一阶段。
```

## 4. 每次改代码前 AI 必须说明什么

学习者可以把下面这条规则发给 AI：

```text
以后你要修改代码前，请先告诉我：
1. 要改哪些文件
2. 每个文件为什么要改
3. 改完后我应该运行什么命令验证
```

这条规则训练的是后端项目里的职责边界意识。

## 5. 每个阶段的完成标准

每个阶段至少满足：

- 能运行
- 能测试
- 能解释
- 能记录

示例：阶段 5 SQLite 对话历史，不只是接口返回成功就算完成。学习者还要能说明：

- 数据库文件在哪里
- `conversations` 表保存了什么
- `/chat` 是什么时候写入数据库的
- `/conversations/{user_id}` 是怎么查历史的
- 为什么不同 `user_id` 的数据不会混在一起

## 6. 如何让 AI 帮助做技术取舍

不要只问：

```text
数据库怎么写？
```

更好的问法：

```text
第一版保存对话历史，SQLite 表结构有哪些设计方案？
请给我 2-3 个选择，说明优缺点，并推荐一个适合新手项目的方案。
```

AI 的价值不是替学习者思考，而是帮助学习者把思考变清楚。

## 7. 日常执行节奏

每次学习或开发可以按这个节奏：

```text
第 1 步：打开 docs/main-quest-progress.md，选择一个小任务
第 2 步：让 AI 解释这个任务的意义
第 3 步：让 AI 带你修改一个很小的部分
第 4 步：运行命令或接口测试
第 5 步：把结果发给 AI 判断
第 6 步：用自己的话总结这一步
第 7 步：在任务表里打勾
```

## 8. 新会话启动提示词

每次换一个新的 AI 会话，可以先发这段：

```text
我正在做 Tutor Agent API 项目。

这是一个训练型后端 API 项目，目标是通过开发 AI 学习辅导 Agent 来学习后端开发。

请你担任我的后端开发导师和结对编程伙伴。

协作规则：
1. 不要一次性帮我生成完整项目。
2. 每次只带我完成 1-3 个小任务。
3. 写代码前先告诉我要改哪些文件、为什么改、怎么验证。
4. 每一步都先解释为什么做，再给命令或代码。
5. 我执行后会把结果发给你，你再判断下一步。
6. 如果我遇到报错，请先帮我分类定位，不要直接猜答案。
7. 每个阶段结束后，请用问题检查我是否真的理解。

当前阶段是：
当前任务是：
```

## 9. 协作底线

- 不一次性生成整个大项目
- 不跳过运行验证
- 不复制自己完全看不懂的代码
- 不把 API key 发给 AI
- 不把所有代码都堆进 `main.py`
- 不为了看起来高级提前引入复杂架构
- 不在没理解前进入下一阶段

这套协作方式的目标是让 AI 帮助学习者加速，但不夺走学习者的成长过程。
